## Complete Observations: C-FLAT Control-Flow Attestation Validation

This summary provides a technical consolidation of the observations made during the validation of your C-FLAT implementation using the Syringe Pump benchmark.

### 1. Architectural Integrity & Design

* 
**Path-Level Hashing:** The system captures the program's runtime behavior by creating a cumulative hash (fingerprint) of every basic block transition.


* 
**Loop Measurement Isolation:** To prevent "path explosion," loops are treated as sub-programs with their own measurement chains, separate from the global cumulative hash.


* 
**Measurement Engine Isolation:** The tracking occurs within the ARM TrustZone "Secure World," ensuring that even if the application is compromised, the attestation report cannot be tampered with.



### 2. Physical Process & Data Integrity

The most critical observation is the translation of physical movement into verifiable software counters:

* 
**Motor Step Verification:** For Command "10" (), the motor loop recorded exactly **68 iterations**. For Command "20" (), it recorded **136 iterations**.


* 
**Linear Scaling:** As demonstrated by your "Input: 100" test, the iterations jumped to **682**, confirming that control flow scales linearly with the calculated physical requirement.


* 
**Non-Control-Data Detection:** Because these counts are reported in the final `Auth` response, the verifier can detect if a variable (like fluid volume) was maliciously changed even if the code itself remains unmodified.



### 3. Security Defense Performance

The logs confirm that the implementation is actively monitoring for high-level runtime attacks:

* 
**ROP/JOP Mitigation:** By performing call-return matching and indexing edges, the system prevents attackers from stitching together benign code gadgets in unauthorized sequences.


* 
**CFG Enforcement:** Any jump to an address not present in the original Control-Flow Graph (CFG) results in an unrecognized final hash, alerting the verifier.


* 
**Data-Oriented Exploits:** The system bridges the gap left by standard Control-Flow Integrity (CFI) by reporting the actually executed path, allowing detection of unauthorized yet valid paths.

Here are the detailed observations on how this works and what it means for the "whole program" attestation:

### 1. Final Hash per Command vs. Program

In a standard C-FLAT deployment, the system computes a cumulative authenticator () for a specific **execution path**.

* **Your Setup:** In your `main` function, you call `process_command()` for each input ("10", "+", etc.). Each call to `process_command()` triggers `cflat_finalize_and_print()`.
* **The Result:** This means you are effectively "resetting" the attestation window for every command. The `Final Hash` you see is the fingerprint of the logic executed **between the start and end of that specific command**.


* **Cumulative Alternative:** If you only called `finalize` once at the very end of the entire program, the `Final Hash` would indeed represent the sum of all operations ("10" + "+" + "20" + "+"). However, that would make debugging and verification much harder.

### 2. Why the Hash is Unique per Command

Each command you enter triggers a different branch in the Syringe Pump's logic:

* 
**"10" (Set Quantity):** This executes the `toUInt` logic and updates the `mLBolus` variable.


* 
**"+" (Move Syringe):** This executes the `bolus()` function, triggers the motor GPIO pins, and iterates through the motor stepping loop.


* 
**Observation:** Because these paths use different basic blocks and different sequences of instructions, the cryptographic hash chain produced in the Secure World results in a unique value for each distinct path.



### 3. Summary of Observations (The "Golden Record")

Based on our analysis of your implementation, here is the complete set of observations for your reference:

#### **A. Basic Block Integrity**

* Every basic block transition (node entry) is intercepted by the **Runtime Tracer**.

* Upon exit, the global hash resumes, and the report adds a **Loop Record** containing the `Loop ID`, `Path Hash`, and `Iteration Count`.
